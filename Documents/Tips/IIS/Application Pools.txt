■リサイクル前の要求は、リサイクル後も動作するか？
○テスト方法
1. サーバー側の処理に 1 分ほどを要する Web サイトを作成し、IIS でホストする
2. ブラウザーでその Web サイトにアクセスする
3. IIS マネージャーでアプリケーション プールをリサイクルする
4. ブラウザーで処理が完了すれば Yes、中断されれば No

○テスト結果
Yes

■停止前の要求は、停止後も動作するか？
○テスト方法
1. サーバー側の処理に 1 分ほどを要する Web サイトを作成し、IIS でホストする
2. ブラウザーでその Web サイトにアクセスする
3. IIS マネージャーでアプリケーション プールを停止する
4. ブラウザーで処理が完了すれば Yes、中断されれば No

○テスト結果
No

■仮想ディレクトリの物理パスの変更前の要求は、変更後も動作するか？
(つまり、アプリケーションのバージョンアップはシームレスに実施できるか？)
○テスト方法
1. サーバー側の処理に 1 分ほどを要する Web サイトを作成し、「1.0」「2.0」フォルダー (物理) に配置する。
   「1.0」フォルダーを IIS でホストする
2. ブラウザーでその Web サイトにアクセスする
3. IIS マネージャーで仮想ディレクトリの物理パスを「2.0」に変更する
4. ブラウザーで処理が完了すれば Yes、中断されれば No

○テスト結果
Yes
直後に別の要求としてアクセスすると、2.0 が動作する。

■セッション
セッションは、ASP.NET 状態サービスなどに格納していれば、リサイクル後やバージョンアップ後も継続して利用可能。
